# Standardised Cohort Analysis

The Social Investment Agency (SIA) has designed a repeatable analysis that uses IDI data to contribute to the knowledge that organisations hold about what works and for whom.

The analysis is designed to provide descriptive and comparative insights into the experiences of different groups of people to support operational decision-making and service design. As a repeatable analysis it provides a standardised process that can be applied to different groups with minimal marginal effort, enabling the delivery of more insights for the same effort.

This repository contains the source code for our repeatable process. The corresponding documentation can be found on our website [here](https://www.sia.govt.nz/publications/Standardised-Cohort-Analysis-Methodology).

## Designed for ease of use

The analysis has been designed to reduce the effort required of analysts to setup and run a new analysis. Detailed instructions are found in the user guidance, in the documentation folder. We have found that once analysts are familiar with the process, a full analysis can be setup within a couple of hours and run without continuous supervision.

## Built on ADAPT

The Accelerated Data Analysis & Pipeline Toolkit (ADAPT) is an analytics toolkit developed by the Social Investment Agency (SIA) to accelerate the delivery of high quality insights. This standardised analysis uses these tools for the core processing and automation. This saves analyst times and ensures consistency.

Researchers seeking to adopt this standardised process are advised to first familiarise themselves with ADAPT. The key resources are the [user guide](https://www.sia.govt.nz/publications/ADAPT-User-Guide) and the [code repo](https://github.com/nz-social-investment-agency/ADAPT).

## Getting started

We recommend the following process for starting with our standardised analysis:
1. Install ADAPT and confirm it is working as per its user guide
2. Copy the Standardised Analysis into your IDI project folder
3. Configure the settings
4. Run the definitions to ensure the underlying data is ready
5. Run the test cohort

The user guidance in the documentation folder covers key details for these steps.
